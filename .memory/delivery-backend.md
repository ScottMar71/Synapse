# Delivery Backend

## Active Backend

`local-files`

## Model

- Keep `outcomes/` as the shared source of truth for product, design, and architecture intent.
- Track engineering execution in repo-local `tickets/*.md` files.
- Link tickets back to their owning outcomes or initiative overview docs.

## Expectations

- Use `tickets/_template.md` as the default structure for new implementation tickets.
- Add `tickets/<initiative>/README.md` when multiple tickets need shared sequencing or coordination notes.
- Keep durable architecture and workflow handoffs in repo docs rather than only in ticket notes.
